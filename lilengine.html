Postmortem

This project appears very simple and limited, but the groundwork for extensibility is laid. Our engine, particularly the saving and loading JSON feature, is very data-oriented. The engine can be played without the editor by calling the play function with a game's JSON file. We were able to include a variety of design patterns as well, such as the singleton pattern with our resource manager and the factory pattern with our component factory.

If we had more time to work on this engine, there are a lot of improvements we could make. More built-in components would allow for more easy functionality in the editor, and are pretty easy to implement. The best component to add would be some sort of generic script component, that allows Python scripts to be attached to GameObjects, rather than implementing Python scripting on the entire engine. The TileMap component could also be improved to work better and work in our editor. The editor could also have the game view, editor menu, and GameObject configuration all in one window with more time. It would also be good to put time into proper scene switching, for more complete games.

It would likely be a lower priority since our engine is too small for major performance issues right now, but there are a lot of places to improve the engine's efficiency. For example, there are a number of inefficient calls to GetComponentOfType, and several Component Update calls that could be reworked to not be called every update tick.
